Escribe una reflexión de 8 a 10 líneas respondiendo:

¿Por qué conectar el frontend con el backend ayuda a que un mini-sitio comunitario mantenga información dinámica, actualizable y útil para la comunidad?

En tu respuesta menciona al menos cinco de estas ideas:

- integración Frontend-Backend
- consumo de la API propia
- mensajes dinámicos
- frontend
- backend
- API
- JSON
- fetch
- async
- await
- render dinámico
- comunicación responsable
- revisión de fuentes
- consistencia comunicativa
- moderación editorial básica
- participación ciudadana
- cultura de paz

Respuesta:
Conectar el frontend con el backend ayuda mucho porque permite que el mini-sitio comunitario tenga información dinámica y más fácil de actualizar. 
El frontend es la parte visual que ven las personas en el navegador, mientras que el backend es la parte que guarda los datos y responde las solicitudes. 
Gracias a la integración Frontend-Backend, la página puede consumir la API propia del proyecto para mostrar mensajes dinámicos, calendarios y resúmenes sin escribir todo manualmente en el HTML.

El frontend usa funciones como fetch junto con async y await para pedir información al backend. Después, los datos llegan en formato JSON y aparecen automaticamente en la página.
 Esto ayuda a mantener la información organizada y actualizada para toda la comunidad.  También es importante porque facilita la comunicación responsable, 
 la revisión de fuentes y la consistencia comunicativa entre los mensajes publicados.
  Además, si se necesita cambiar un aviso, corregir información o agregar nuevos mensajes, solo se actualiza el backend y automáticamente el mini-sitio muestra los cambios. 
 Todo esto ayuda a fortalecer la participación ciudadana y promover una cultura de paz en los espacios digitales comunitarios.
